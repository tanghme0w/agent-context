## 一、产品线定位

Rokid 有两条产品线，对应两套操作系统：

| 产品线 | 操作系统 | 代表产品 | 定位 |
|--------|----------|----------|------|
| AR 眼镜 | YodaOS-Master | AR Studio / AR Lite | 空间计算、3D 交互 |
| **AI 眼镜** | **YodaOS-Sprite** | **Rokid Glasses** | **全天佩戴、AI 助手、轻量化** |

> **YodaOS-Sprite** 官方定义：一套为全天候佩戴使用而打造的智能眼镜操作系统，通过对芯片、驱动、系统、应用和产品场景的深度优化和打磨，实现了续航、性能和功能三者之间的平衡。YodaOS-Sprite 以自然无感化设计为核心理念，消除科技的存在感，让智能眼镜成为用户日常生活的自然延伸。

**本文聚焦 YodaOS-Sprite（AI 眼镜 / Rokid Glasses）这条线。**

## 三、开发知识体系总览

YodaOS-Sprite 的开发体系分为 **两条开发路径**：

```
                    YodaOS-Sprite 开发体系
                           │
              ┌────────────┴────────────┐
              │                         │
        移动端应用开发              眼镜端裸机开发
         (CXR-M SDK)              (CXR-S SDK)
              │                         │
    手机作为主控端                 眼镜独立运行
    眼镜作为外设端                 直接访问硬件
              │                         │
    ┌─────────┴─────────┐        ┌──────┴──────┐
    │   数据/音视频通信   │        │  按键管理    │
    │   AI 场景控制      │        │  IMU 传感器  │
    │   眼镜端显示控制   │        │  Camera 控制 │
    │   文件同步传输     │        │  音频管理    │
    │   传感器数据获取   │        │  自定义应用  │
    └───────────────────┘        └─────────────┘
```

### 3.1 路径一：移动端应用开发（CXR-M SDK）

**官方描述：**
> 开发者通过移动端开发，可以与眼镜端进行交互，实现数据通信；音视频通信；控制眼镜端显示自定义页面；也可以在眼镜端开发自定义应用，并通过自定义指令的形式实现与移动端的数据通信。从而构建高效敏捷的 AI 应用。

**当前版本：** v1.0.9（2026-03-01 更新）

**平台：** 仅 Android

**开发模式：** 手机端作为主控，眼镜端作为协作外设

### 3.2 路径二：眼镜端裸机开发（CXR-S SDK）

**官方描述：**
> Rokid Glasses 支持裸机开发，开发者可以通过裸机开发，直接在眼镜端开发自定义应用，并可以管理眼镜端的按键、IMU、Camera 等资源。开发者可以根据自己的应用场景构建眼镜端独立运行的应用。

**当前版本：** v0.0.1（2026-03-01 更新，全新发布）

**开发模式：** 眼镜端完全独立运行，直接调用硬件资源

**CXR-S 与 CXR-M 的关系：**
> CXR-S SDK 是运行在 YodaOS-Sprite 上的眼镜端开发工具包，专注于帮助开发者直接在 Rokid Glasses 上创建独立应用。它不仅提供访问眼镜端数据通道的能力，还能与移动端的 CXR-M SDK 建立双向通信，支持自定义协议和指令传输。通过 CXR-S SDK，开发者可以更深入地调用硬件资源，释放设备潜能，打造低延迟的智能体验。

## 五、Q&A（官方常见问题）

**Q1: CXR-M SDK 的主要功能包含哪些？**
A: 使用 CXR-M SDK 开发的移动端应用，可以和 Rokid Glasses 进行通信，获取 Glasses 上的音频、视频，也可以自定义 Rokid Glasses 上的已有的场景的实现，在与 Rokid CXR-S SDK 配合时，也可以进行自定义指令的交互。

**Q2: CXR-M SDK 目前支持哪些设备使用？**
A: 目前 CXR-M SDK 只提供了 Android 移动设备的 SDK。

**Q3: CXR-S SDK 的主要功能包含哪些？**
A: CXR-S SDK 提供了 YodaOS-Sprite 上数据通道的访问能力，可以将自定义的指令通过数据通道传递给 CXR-M SDK。

**Q4: 仅使用 CXR-M SDK 开发移动端应用时，需要打开 Rokid Glasses 的开发者权限吗？**
A: 不需要。

## 八、注意事项

1. **AR 线 vs AI 线**：UXR SDK（Unity 开发）和 AR Studio 属于 YodaOS-Master（AR 线），与 Rokid Glasses（Sprite 线）是完全不同的开发体系
2. **CXR-M vs CXR-S**：CXR-M 是手机端 SDK，CXR-S 是眼镜端 SDK，两者可以独立使用，也可以配合使用
3. **CXR-S 刚发布**：眼镜端裸机开发 SDK v0.0.1 于 2026.03.01 才首次发布，目前仅包含按键和音频，后续版本会增加更多硬件接口
4. **详细 API 文档**在 custom.rokid.com 上，需通过 developerdoc.rokid.com/sdk 页面的"查看文档"按钮跳转访问
5. **灵珠平台**（rizon.rokid.com）是 Rokid 的 AI 能力平台，与 SDK 开发平台是独立的入口
